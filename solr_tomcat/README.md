## Solr delete index

http://localhost:8983/solr



If you get a IndexWriter is Closed

* stop tomcat, solr
* remove file write.lock
* start tomcat, solr

And run the document delete all again.

Method 1: Using the Solr Admin UI
You can clear the index directly from the browser administration panel.

🔹 1. Open your browser and navigate to your Solr Admin dashboard (typically http://localhost:8983/solr/).

🔹 2. Select your specific core or collection from the Core/Collection Selector dropdown menu on the left side panel.

🔹 3. Click on the Documents tab.

🔹 4. Change the Document Type dropdown setting to XML.

🔹 5. In the Document(s) text area box, input the following text exactly:

```xml
<delete><query>*:*</query></delete>
```
🔹 6. Ensure that the Commit Within value is specified (e.g., 1000) or pass a clean hard commit parameter if desired.

🔹 7. Click the Submit Document button. The right-hand panel will display a success block message matching status: 0

## Tomcat logs

1. catalina.out (Standard Output & Error)

* This is the most critical file for troubleshooting initial server startup crashes or unhandled application panics.

2. catalina.YYYY-MM-DD.log (Engine Log)

* Log messages generated by Tomcat’s internal components (such as startup initializations, connector bindings, servlet deployments, and shutdown commands) routed through JULI.

3. localhost.YYYY-MM-DD.log (Host App Log)

* Uncaught exceptions inside your applications, listener failures, and initialization errors within the web application scope. If a servlet fails to initialize or a database connection pool configuration fails inside your app, the trace is usually logged here.

4. localhost_access_log.YYYY-MM-DD.txt (Access Log)

* Incoming client request vectors, including client IP, timestamp, HTTP request method (GET, POST, etc.), requested URI paths, HTTP status codes (200, 404, 500), and bytes sent
